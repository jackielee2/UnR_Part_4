-----------------------------------
0]	개요
-----------------------------------

	-	액터의 이동과 회전은 매 프레임마다 반복적으로 수행됨.
	-	초당 프레임 수( FPS, Frame Per Second )는
		플랫폼의 종류나 퍼포먼스에 따라 다름.
	-	프레임당 일정거리를 이동하게 했을때 결과.

					|	높은 FPS					|	낮은 FPS
		------------------------------------------------------------------------
		이동 거리	|	상대적으로 먼 거리 이동	|	상대적으로 적은 거리 이동.
		------------------------------------------------------------------------
		속도			|	상대적으로 빠름			|	상대적으로 느림.
		------------------------------------------------------------------------

		예)	초속 2000으로 이동 하는 액터의 1초후 위치..
			
			100 FPS 디바이스		=	2000 * 100
			50 FPS 디바이스		=	2000 * 50
		


-----------------------------------
1]	변수 추가.
-----------------------------------

	-	개요.
	
		-	헬기의 이동과 회전에 필요한 변수 추가.
			

	-	실습.

		1)	[ BP_Apache ] 열기.

		2)	[ 내 블루프린트 ] > [ 변수 ] > [ + 버튼 ] 클릭.

		2)	변수 추가.

			1)	이동 관련 변수 추가
				-	이름		:	[ Move Speed ]
					타입		:	[ Float ]
					초기값	:	[ 3000 ]
								-	초속 30m 이동( 기본 단위 cm )
								-	컴파일 하면 초기값 설정 가능.

				-	이름		:	[ Rot Speed ]
					타입		:	[ Float ]
					초기값	:	[ 90 ]
								초속 90도 회전.



-----------------------------------
2]	이벤트 그래프 작성.
-----------------------------------
	
	-	개요.

		-	상하 이동 키를 누르면 헬기가 전후로 이동하는 기능 작성.
		-	원리
			-	동일한 PC에서도 FPS는 항상 일정하지 않으므로
				이전 프레임과 현재 프레임의 시간차이( [ deltaTime ] )을 구하고
				여기에 [ 속도 ]와 [ 입력된 키의 정보 ( -1.0 ~ 1.0 ) ]를 곱하여
				[ 속도 벡터 ]를 만든후 현재 액터의 [ 위치 ]에 누적.

			-	[ 액터의 위치 ] = [ 액터의 위치 ] + ( [ deltaTime ] * 속도 * 입력 키 정보 )

	
	
	-	실습.
		
		-	[ BP_Apache ] > [ 이벤트 그래프 ] 참고.
